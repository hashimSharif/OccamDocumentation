Detailed Description

The SpecializationTable is a repository of specializations. If a specialized version of a function has been created, specific to a set of arguments, there is no need to create another version, if the function is called with exactly the same arguments. Therefore before a specialized copy of a function is required, it is looked up in the SpecializationTable. If a function corresponding to the specialization exists, it is retrieved from the table, otherwise previrt::specializeFunction is called to create a new specialized version with respect to the usage.

Returns all the specializations against a function that are at least as strict as specified by the scheme (calling SpecializationTable::getSpecializations). A specialization is as strict as the other if it specializes no more arguments as compared to the other. For example, consider a function with three arguments, and the scheme specifies only the first argument as specialize-able (NON-NULL value), then all specializations that specialize the first argument, or the ones specializing none are returned. Essentially we want only the matching specializations, therefore in the files StrategySpecializer.cpp line 167, SpecializationTable::refines is called once again this time with inverting the specScheme (to specialize with respect to), and the specialization scheme of the arguments, thereby only functions that take exactly the same arguments can be extracted from the Specialization Table
