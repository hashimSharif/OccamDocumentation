50  /*! SpecializerPass is extending the functionality of llvm:: ModulePass to implement partial evaluation within a module (intra module
51  specialization). The procedure of partial evaluation involves:
52 
53  1) Iterating over all the call-sites in all the functions in the Module.
54 
55  2) Identifying the call-sites that pass constant arguments.
56 
57  3) Based on the specific usages, specialized versions of functions are created, if they are not found in the previrt::SpecializationTable
58 
59  4) The SpecializerPass::runOnModule routine creates these specialized versions and adds them to the Module
60 
61  5) The call-sites are replaced by, calls to the specialized version corresponding to the specific usage.
62 
63  SpecializerPass is registered with the internal PassManager database by the name 'Ppeval'. It accepts an optional command line argument
64  'Ppeval-opt'. If 'Ppeval-opt' is specified, the SpecializerPass::runOnModule function applies llvm optimization onto the newly created
65  specialized functions.
66  */
