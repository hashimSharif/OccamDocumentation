Detailed Description

The SpecializeArguments class extends the functionality of the ModulePass to incorporate specialization of command line arguments into the program. In case the arguments to a program are not expected to change, the efficiency and security of the program by applying partial evaluation can be improved, by directly embedding the values of the program arguments into the main function body.

RegisterArguments is a class that is derived from SpecializeArguments and is a wrapper above the class. RegisterArguments is registered as a pass in the PassManager internal database by the name 'Parguments'.

bool previrt::SpecializeArguments::runOnModule ( llvm::Module &  )
virtual

The runOnModule method specializes the main function of the Module M, given the command line arguments in the input file to the pass. The steps of the procedure are :

1) Get a pointer to the main function of the Module.

2) Create an empty new main with the same function type and linkage as the original main.

3) The name of the original main is taken and assigned to the new main function.

4) Create a Basic Block and add it to the new main function.

5) For all arguments in the argv array (class data member) create a global variable.

6) In the new main function, in the very first Basic Block, add an call to the original main function along with the global variable arguments.

7) Set the linkage of the original main function to GlobalValue::PrivateLinkage and a function attribute of Attribute::AlwaysInline.

8) Apply an in-liner pass, to make embed all in-line function calls, into the function body.

9) Thereby essentially the new main function will preserve the semantics of the body of the original function, just replacing the references to argv and argc with references to global variables with static values.
