76  /*!
77  The \e specializeOn function is a virtual function, with separate implementations in the derived classes
78  ConservativeSpecPolicy, AggressiveSpecPolicy
79  and RecursiveGuardSpecPolicy. The detailed descriptions of each implementation are as follows:
80 
81 
82  ConservativeSpecPolicy::specializeOn , AggressiveSpecPolicy::specializeOn detailed description:
83 
84  The specializeOn method, returns a Value **. Value ** is essentially a array defining the specialization policy against
85  each argument. If an argument is a constant and is can be partially evaluated into the function body, the argument is returned in the array.
86  For arguments cannot be partially evaluated into the function body, their corresponding index in the Value* array is NULL.
87 
88  RecursiveGuardSpecPolicy::specializeOn detailed description:
89 
90  * The function guards the specialization of recursive functions. It Returns a policy array consisting of a policy against every argument to the function.
91  * The array contains NULL values against arguments that cannot be specialized and NON-NULL values against
92  * arguments that are to be specialized.
93  * The function in turn calls the \e specializeOn method of the delegate class member.
94  * Delegate can be an object of two possible classes: ConservativeSpecPolicy and AggressiveSpecPolicy.
95  * If the function is a recursive function, NULL is returned, as recursive functions
96  * cannot be specialized. (specialization could result in a large number of specialized versions of the recursive function)
97 
98  @param F The pointer to the Function being specialized
99  @param begin The iterator to the start of the argument list
100  @param end The iterator to the end of the argument list.
101  */
