Detailed Description

The OccamPass Class performs dead code elimination in the module. By default all functions in llvm are external linkage(they can be called from an external module), hence it is necessary to be certain that a function is not called from an external module before it is pruned. All functions not called externally, are modified to internal linkage, and an llvm dead code elimination pass is applied.

The OccamPass class is registered as a pass with the PassManager internal database with the name 'Poccam'. File names (required argument) followed by the flag 'Poccam-input' are previrt::ComponentInterface objects that are a wrapper above the Google protocol buffer previrt::proto:ComponentInterface that contain a list of functions and references that are being used externally. The list of externally called functions and references is required so that they are not incorrectly eliminated from the module.
